创意名称 + 创意介绍

把互动叙事创作从"写代码"还原为"讲故事"。

想解决什么问题？

当前制作互动叙事类作品（互动小说、性格测试、剧情向小游戏）门槛极高——需要编程基础、需要理解状态机和流程图逻辑。普通创作者（策划、文案、运营）只能依赖技术人员实现，沟通成本高、迭代慢。

为什么会想到做这个？

灵感来源于为游戏社区制作"本命门派性格测试"类互动内容。这类内容需求量大、形式固定，但每次都要从零开发。我意识到：如果把"节点编排 + 变量系统 + 人格向量计算"抽象成一个通用可视化引擎，任何人都能像画流程图一样创作互动叙事作品。

大概是什么产品？

一个基于浏览器的 Web 应用。左侧是资产面板，中间是可视化节点画布（拖拽连线），右侧是属性配置面板。创作者通过拖拽节点、连线、配置参数即可完成互动剧情编辑，点击"试玩"即可实时体验，最终可导出为独立 JSON 或发布到云端。

当前痛点

  • 现有工具要么太专业（Unity/Unreal 可视化脚本），要么太简单（只能做线性选择题），缺乏"可视化编程 + 变量系统 + 人格计算"兼顾的中间方案。
  • 传统开发模式下，文案改一个选项都要找开发改代码，迭代效率极低。
  • 性格测试类工具只能做静态评分，无法实现"向量空间最近邻匹配"等高级人格算法。

创意亮点

技术差异化让 Zorron Engine 在同类工具中脱颖而出。

三维人格向量空间

独创 X（处世）/ Y（立场）/ Z（性情）三维坐标系，每个选项携带向量偏移。通过"碎片收集 + 向量累加 + 最近邻匹配"算法，实现比传统评分更细腻、更可解释的人格判定。

X轴：处世（出世 ↔ 入世）
Y轴：立场（混沌 ↔ 守序）
Z轴：性情（感性 ↔ 理性）
